# Existing Solutions

## Purpose

Этот документ фиксирует исследование существующих решений для Этапа 0 ViktorOS.

Цель документа — понять, какие части ViktorOS уже могут быть собраны из готовых инструментов, а какие требуют собственной разработки.

---

## Stage Context

Этап 0 появился после Iteration 001 как необходимость проверить проект до начала собственной разработки.

На этом этапе ViktorOS не должен преждевременно превращаться в архитектуру, набор агентов или Markdown-архив.

Сначала необходимо понять:

- какие задачи уже решены существующими продуктами;
- какие решения можно использовать как готовые элементы;
- где существующие решения не закрывают задачу ViktorOS;
- какая часть проекта действительно должна быть собственной.

---

## Research Questions

Исследование должно ответить на четыре вопроса:

1. Что уже реализовано другими проектами?
2. Что можно использовать без собственной разработки?
3. Чего действительно не хватает?
4. Какая часть должна стать уникальной для ViktorOS?

---

## Evaluation Criteria

Каждое решение оценивается по следующим критериям:

- какую проблему оно решает;
- насколько оно близко к ViktorOS;
- можно ли использовать его без собственной разработки;
- какие ограничения у него есть;
- какую часть ViktorOS оно может закрыть;
- что остается нерешенным.

## Research Procedure

Для каждой категории используется один и тот же порядок:

1. Описать задачу категории.
2. Найти 3–7 существующих решений.
3. Кратко оценить каждое решение по критериям.
4. Сформулировать вывод по категории.
5. Зафиксировать, какую часть ViktorOS эта категория может закрыть.
6. Зафиксировать, что остается нерешенным.

## Category 001: Personal Knowledge Management systems

Статус: started.

### Purpose

Проверить, могут ли существующие системы управления личными знаниями использоваться как основа для хранения, редактирования и переноса модели человека.

### Candidate Solutions

Первичные кандидаты для исследования:

- Obsidian;
- Logseq;
- Notion;
- Anytype;
- Capacities;
- Reflect;
- Tana.

### Initial Hypothesis

PKM-системы могут быть полезны как интерфейс и временное хранилище знаний, но, вероятно, не являются полной основой ViktorOS.

Главное ограничение: большинство PKM-систем организуют информацию для человека, но не создают независимую модель человека, которую любой ИИ может стабильно использовать без повторного восстановления контекста.

### Open Questions

- Какая PKM-система лучше всего подходит для локального хранения модели?
- Какая система лучше всего подходит для связи заметок и контекстов?
- Какая система проще всего интегрируется с ИИ?
- Какая система обеспечивает максимальную переносимость данных?
- Где заканчивается польза PKM и начинается необходимость собственной структуры ViktorOS?
