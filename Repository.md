# Repository

## Purpose

Этот документ определяет правила работы с репозиторием ViktorOS.

---

## Principles

- GitHub является единственным источником истины (Single Source of Truth).
- Любые документы, память модели, история чата и локальные копии являются вторичными относительно GitHub.
- Перед любым изменением файл всегда сначала перечитывается из GitHub.
- Все документы ViktorOS хранятся в формате Markdown.
- Разработка ведется только в исходном Markdown.
- Режим предпросмотра используется только для проверки итогового результата.
- Каждый документ должен иметь одну четкую ответственность.
- Commit должен кратко и точно описывать смысл изменения.
- История Git является частью документации проекта.

---

## Workflow

1. Прочитать актуальную версию файла из GitHub.
2. Внести изменения.
3. Проверить структуру документа.
4. Сохранить изменения в GitHub.
5. Создать осмысленный commit.

---

## Workflow Rule

После утверждения любого изменения я автоматически формирую GitHub Task.

Основной чат принимает решения.

GitHub-чат только вносит утвержденные изменения в репозиторий.

---

## GitHub Sessions

Для стабильной работы используется разделение сессий.

- Основной чат предназначен для проектирования, обсуждения и принятия решений.
- Отдельный GitHub-чат используется исключительно для работы с репозиторием.
- После утверждения решения изменения вносятся через GitHub-чат.

---

## Known Infrastructure Limitations

На текущий момент установлены следующие особенности платформы ChatGPT.

- GitHub-инструменты могут быть недоступны в длинных чатах.
- В новом коротком чате GitHub-инструменты стабильно доступны.
- Любое изменение считается выполненным только после успешного commit.
- Любое ограничение платформы должно документироваться и учитываться при проектировании ViktorOS.
